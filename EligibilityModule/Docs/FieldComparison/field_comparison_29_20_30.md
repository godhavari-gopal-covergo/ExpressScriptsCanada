## Summary

- **Fields in all three Record Types (29, 20, 30):** 4 fields (common identification fields)
- **Fields only in Record 29:** 9 fields (address-specific fields)
- **Fields in Record 20 and Record 30 but not in Record 29:** 12 fields (general processing and EFT fields)
- **Fields only in Record 20:** 42 fields (pharmacy and dental sections)
- **Fields only in Record 30:** 16 fields (EHC section)
- **Total unique fields:** 83 fields

## Common Fields (4)
All three record types share the following identification fields:
1. Record Type
2. Carrier ID
3. Group Number
4. Client ID

## Record 29 Specific Fields (9)
Record 29 is a Client Address record that includes address-specific fields:
1. Client Last name
2. Client first name
3. Address1
4. Address2
5. City
6. Province
7. Country
8. Postal Code
9. Filler

**Note:** Record 29 does not include SAS, general processing mode, EFT fields, or any benefit-specific sections (pharmacy, dental, or EHC).

## Fields in Record 20 and Record 30 but not in Record 29 (12)
Both Record 20 and Record 30 share these fields that are not present in Record 29:
1. SAS
2. General Processing Mode
3. Alternate Identification
4. Client Language Flag/Code
5. Client Province Code
6. EFT Account #
7. EFT Route code
8. EFT Effective Dt
9. EFT Termination date
10. Lookup override code
11. Employment enrolment date
12. Filler (general section)

## Record 20 Specific Fields (42)
Record 20 includes Pharmacy and Dental sections with 42 additional fields that are not in Record 29 or Record 30.

## Record 30 Specific Fields (16)
Record 30 includes EHC (Extended Health Care) section with 16 additional fields that are not in Record 29 or Record 20.

## Key Differences

**Record 29 (Client Address):**
- Focus: Address information only
- Purpose: Provides client address details separately from benefit configuration
- Structure: Minimal identification fields + complete address fields
- Not mandatory: Can be sent multiple times after initial client record

**Record 20 (Client with Pharmacy & Dental):**
- Focus: Complete client configuration including pharmacy and dental benefits
- Purpose: Full client record with all processing modes and benefit settings
- Structure: Identification + General + Pharmacy + Dental sections

**Record 30 (EHC Client):**
- Focus: Client configuration for Extended Health Care benefits only
- Purpose: EHC-specific client record
- Structure: Identification + General + EHC sections
